# Phase 2 — Goal & Pacing

**Goal:** Define the game's win/end condition and its subgoal structure for pacing.

---

## Core Concepts to Apply

**Good goals:**
- Let players know immediately what they're trying to achieve
- Create gameplay (meaningful decisions emerge from pursuing them)
- Scale into clear subgoals

**Bad goals:**
- Unclear optimization ("win by being second" — no one knows how to play toward that)
- No gameplay created (just luck, or only one obvious move)
- Anti-patterns: analysis paralysis, false choices, expectation-breaking

**Pacing via subgoals:**
A player should always have a moment-to-moment goal AND a longer-term goal.
Example from basketball:
- Moment-to-moment: score this possession
- Short-term: win the quarter
- Medium-term: win the game
- Long-term: win the season

---

## Questions to Work Through

### 2a. The Main Goal (Win Condition)
> "How does a player win — or how does the game end? Don't overthink it; even a rough idea
> is fine. What does 'done' look like?"

If stuck, offer examples:
- Reach a location first
- Accumulate the most points
- Be the last one standing
- Complete a shared objective before time runs out
- Eliminate opponents

### 2b. Mechanical Representation
> "How would that goal show up physically in the game? Like, what object, counter, card, or
> space represents 'winning'?"

This bridges theme to mechanics. Help them make it concrete.

### 2c. Subgoals
> "What smaller things would a player need to accomplish *on the way* to that main goal?
> These are your subgoals — the stepping stones that give the game its rhythm."

Guide them: "Think of it like levels in a video game. Each level completion is a subgoal.
What's the equivalent in your game?"

### 2d. Pacing Check
> "Roughly how long should a full game take? And within that, how often should something
> exciting happen — a subgoal being hit, a twist, a decision point?"

Use this to sanity-check whether their subgoal structure is dense enough. A 10-minute game
with one meaningful moment is under-paced. A 10-minute game with 20 tense moments
might be exhausting.

### 2e. Goal Quality Check
Before moving on, silently evaluate:
- Does the goal create meaningful decisions, or is it too passive?
- Is it clear enough that a new player would understand it immediately?
- Does it produce interesting subgoals?

If it fails, say: "This goal is interesting, but I want to flag something: [problem].
What if we adjusted it to [suggestion]? That would give players more to think about."
